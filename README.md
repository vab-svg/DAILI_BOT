# Subscription Control Bot

Личный Telegram-бот для контроля подписок и сервисов:
- ежемесячные и годовые подписки
- балансовые сервисы вроде OpenAI, прокси, anti-captcha, мобильной связи
- напоминания о списаниях
- напоминания о низком балансе
- прогноз падения баланса по выбранной модели расхода
- фиксация оплат и пополнений
- ежедневная сводка
- отчёт и история трат

## Важно

Данные хранятся **только в памяти процесса**.
Если бот будет перезапущен, список подписок и история обнулятся.
Это сделано специально под условие: локальное хранение в рантайме без БД.

## Быстрый запуск

### 1. Распакуй архив

### 2. Создай виртуальное окружение

#### Linux / macOS

```bash
python3.13 -m venv .venv
source .venv/bin/activate
```

#### Windows PowerShell

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 3. Установи зависимости

```bash
pip install -r requirements.txt
```

### 4. Подготовь `.env`

Скопируй `.env.example` в `.env` и заполни:

```env
TG_BOT_API_KEY=твой_токен_бота
OWNER_USER_ID=твой_telegram_user_id
TIMEZONE=Europe/Berlin
DAILY_SUMMARY_TIME=09:00
ALERTS_TIME=08:30
SOON_DAYS=14
BALANCE_WARNING_DAYS=3
```

`OWNER_USER_ID` можно оставить пустым, тогда бот не будет ограничен одним аккаунтом.
Для личного использования лучше указать свой Telegram user id.

## Что умеет бот

### Основные команды

- `/start` — старт и меню
- `/add` — добавить подписку
- `/list` — список подписок
- `/soon` — ближайшие списания
- `/topup` — балансовые сервисы, низкий баланс и прогнозы
- `/pay` — отметить оплату или пополнение
- `/setbalance` — обновить текущий баланс
- `/dashboard` — краткая сводка
- `/report` — отчёт за текущий месяц
- `/history` — история последних трат
- `/demo` — создать демо-набор
- `/cancel` — отменить текущий диалог

### Поддерживаемые типы сервисов

1. **Ежемесячная подписка**
   - хранит сумму
   - дату следующего списания
   - напоминание за N дней

2. **Годовая подписка**
   - подходит для доменов, лицензий, хостинга на год

3. **Балансовый сервис**
   - хранит обычную сумму пополнения
   - текущий баланс
   - минимальный порог
   - присылает предупреждение, когда баланс меньше или равен порогу
   - умеет прогнозировать снижение баланса по модели расхода

### Модели расхода для балансового сервиса

1. **Ручной контроль**
   - бот хранит только последний введённый баланс
   - напоминание приходит, когда ты сам обновишь баланс и он будет ниже порога

2. **Фиксированное списание**
   - ты задаёшь сумму одного списания
   - задаёшь период списания в днях
   - бот считает прогнозный текущий баланс от последней точки отсчёта
   - бот показывает дату следующего прогнозного списания
   - бот заранее предупреждает, когда баланс примерно дойдёт до порога

3. **Средний расход в день**
   - ты задаёшь средний дневной расход
   - бот считает прогнозный остаток по дням
   - бот заранее показывает, через сколько дней баланс дойдёт до порога

## Как работает прогноз баланса

Для балансовых сервисов бот использует **последний подтверждённый баланс** как точку отсчёта.
Она обновляется в трёх случаях:
- при создании подписки
- при `/setbalance`
- при пополнении через `/pay`

Если при пополнении отправить `-`, бот прибавит сумму пополнения к текущему прогнозному балансу и запишет сегодняшний день как новую точку отсчёта.
