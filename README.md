# Subscription Control Bot

Личный Telegram-бот для контроля подписок и сервисов.

Что уже умеет:
- ежемесячные и годовые подписки
- балансовые сервисы
- прогноз падения баланса
- фиксация оплат и пополнений
- история трат
- ежедневная сводка
- учёт пользователей

Что добавлено в этой версии:
- редактирование подписок через `/edit` и кнопку **✏️ Редактировать**
- постоянное локальное сохранение в JSON-файл
- гибкие напоминания для обычных подписок: например `7,3,1,0`
- ежедневные напоминания по просроченной подписке до отметки оплаты
- история по каждой подписке через кнопку **🧾 История**
- быстрые кнопки: оплатил, изменить баланс, отложить, пауза, удалить, редактировать

## Хранение данных

Данные теперь сохраняются локально в JSON-файл.
По умолчанию используется:

`subscription_data.json`

Путь можно изменить через `.env` переменную `DATA_FILE`.

## Требования

- Python 3.13.12
- Telegram bot token

## Быстрый запуск

### Linux / macOS

```bash
python3.13 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python main.py
```

### Windows PowerShell

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
python main.py
```

## Настройка `.env`

```env
TG_BOT_API_KEY=твой_токен_бота
OWNER_USER_ID=
TIMEZONE=Europe/Berlin
DAILY_SUMMARY_TIME=09:00
ALERTS_TIME=08:30
SOON_DAYS=14
BALANCE_WARNING_DAYS=3
DATA_FILE=subscription_data.json
```

## Основные команды

- `/start` — старт и меню
- `/add` — добавить подписку
- `/edit` — изменить подписку
- `/list` — показать подписки
- `/soon` — ближайшие списания
- `/topup` — балансовые сервисы и низкий баланс
- `/pay` — отметить оплату или пополнение
- `/setbalance` — обновить баланс вручную
- `/dashboard` — сводка
- `/report` — отчёт за текущий месяц
- `/history` — общая история трат
- `/users` — кто пользуется ботом
- `/demo` — демо-данные
- `/cancel` — отмена текущего диалога

## Как работают новые напоминания

Для обычной подписки ты задаёшь список дней через запятую.

Пример:
- `7,3,1,0` — напомнить за 7 дней, за 3 дня, за 1 день и в день списания
- `3,1,0` — только ближе к списанию

Дополнительно можно включить режим:
- **напоминать каждый день после просрочки, пока не отметишь оплату**

## Быстрые действия у карточки подписки

### Для обычной подписки
- **💸 Оплатил**
- **🧾 История**
- **✏️ Редактировать**
- **⏰ Отложить**
- **⏯ Пауза / Возобновить**
- **🗑 Удалить**

### Для балансового сервиса
- **💸 Пополнил**
- **🪫 Изм. баланс**
- **🧾 История**
- **✏️ Редактировать**
- **⏰ Отложить**
- **⏯ Пауза / Возобновить**
- **🗑 Удалить**

## История по подписке

Кнопка **🧾 История** показывает:
- сколько было операций по конкретной подписке
- сколько потрачено по ней за текущий месяц
- последние операции по ней

## Примечание

JSON-хранилище локальное. Если среда деплоя полностью пересоздаётся, файл может исчезнуть.
Для Railway это значит, что при новом деплое локальный файл обычно не считается надёжным постоянным хранилищем.
Для полностью стабильного хранения в будущем лучше перейти на SQLite/Postgres.
